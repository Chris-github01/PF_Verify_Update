# DocRaptor PDF Export Integration

Complete DocRaptor integration for professional PDF generation from HTML reports with consistent A4 layout.

## Features Implemented

### ✅ A) Download PDF UX

**Location:** Contract Manager > Site Handover tab

- **Two buttons for each report type:**
  - **HTM Export**: Opens browser print dialog (existing functionality)
  - **Download PDF**: Generates professional PDF via DocRaptor

- **Loading States:**
  - Buttons show "Generating..." while PDF is being created
  - Buttons are disabled during generation to prevent duplicate requests

### ✅ B) PDF Render Mode

**Files:**
- `src/lib/reports/pdfStyles.ts` - Centralized PDF CSS styles
- `src/lib/reports/modernPdfTemplate.ts` - Updated with renderMode support

**Print-Safe CSS:**
```css
@page {
  size: A4;
  margin: 12mm 12mm 14mm 12mm;
}

* {
  print-color-adjust: exact;
  -webkit-print-color-adjust: exact;
}

.avoid-break {
  break-inside: avoid;
  page-break-inside: avoid;
}

.page-break {
  break-before: page;
  page-break-before: always;
}
```

**Image & Table Handling:**
- Images: `max-width: 100%; height: auto;`
- Tables: Prevent row splits with `break-inside: avoid`
- Cards/Sections: Wrapped in `.avoid-break` class

### ✅ C) Server-Side DocRaptor Integration

**Edge Function:** `supabase/functions/generate_pdf_docraptor/index.ts`

**Functionality:**
1. Receives HTML content from frontend
2. Injects print-safe CSS styles
3. Adds headers/footers using @page CSS
4. Calls DocRaptor API with authentication
5. Returns PDF as `application/pdf` with proper filename
6. Automatic fallback on error

**Environment Variable:**
```
DOCRAPTOR_API_KEY=your_api_key_here
```

**Test Mode:**
- Development: `test: true` (uses DocRaptor test credits)
- Production: `test: false` (uses paid credits)

### ✅ D) Headers & Footers

**Configured via CSS @page:**

```css
@page {
  @top-left {
    content: "Project Name | Contract Number";
    font-size: 9pt;
    color: #6b7280;
  }

  @top-right {
    content: "2025-12-20 | Page " counter(page) " of " counter(pages);
    font-size: 9pt;
    color: #6b7280;
  }

  @bottom-center {
    content: "Generated by VerifyTrade";
    font-size: 8pt;
    color: #9ca3af;
  }
}
```

**Dynamic Content:**
- Project name and contract number from database
- Current date (YYYY-MM-DD format)
- Page numbering (Page X of Y)

### ✅ E) Reliability & Error Handling

**Automatic Fallback:**
```typescript
if (result.success && result.blob) {
  downloadPdfBlob(result.blob, filename);
} else {
  alert(`PDF service unavailable. Falling back to browser print.`);
  fallbackToHtmExport(result.fallbackHtml, filename);
}
```

**Error Logging:**
- Server-side errors logged with reportId
- User-friendly error messages
- HTM export always available as fallback

**Timeout Handling:**
- DocRaptor timeout: 300 seconds
- Edge function timeout: 120 seconds (default)

## Setup Instructions

### 1. Get DocRaptor API Key

1. Sign up at [https://docraptor.com](https://docraptor.com)
2. Get your API key from the dashboard
3. Test account provides free test documents

### 2. Configure Supabase Edge Function

Add the API key to your Supabase project:

```bash
# Using Supabase CLI
supabase secrets set DOCRAPTOR_API_KEY=your_api_key_here

# Or via Supabase Dashboard
# Settings > Edge Functions > Secrets
# Add: DOCRAPTOR_API_KEY = your_api_key_here
```

### 3. Deploy Edge Function

```bash
# Deploy the DocRaptor function
supabase functions deploy generate_pdf_docraptor
```

### 4. Test the Integration

1. Navigate to Contract Manager > Site Handover
2. Click "Download PDF" button
3. PDF should download automatically
4. If it fails, HTM export will open as fallback

### Frontend Call


## Troubleshooting

### PDF Not Downloading

1. Check browser console for errors
2. Verify DOCRAPTOR_API_KEY is set in Supabase
3. Check edge function logs in Supabase dashboard
4. Try HTM export to verify HTML generation works

### Headers/Footers Not Showing

- Ensure page margins are set correctly (12mm minimum)
- Verify @page CSS is in the `<head>` section
- Check DocRaptor documentation for @page support

### Images Not Rendering

- Images must be publicly accessible URLs
- Use full URLs (not relative paths)
- Verify image URLs work in browser

### Table Rows Split Across Pages

- Wrap tables in `.avoid-break` class
- Apply `break-inside: avoid` to `<tr>` elements
- Consider splitting long tables into multiple pages
