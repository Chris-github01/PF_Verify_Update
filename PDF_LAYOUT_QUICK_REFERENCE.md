# PDF Layout Quick Reference

## ✅ DO - Correct Patterns

### Page Margins
```css
@page {
  size: A4;
  margin: 18mm 14mm 18mm 14mm;  /* Consistent across all PDFs */
}
```

### Page Sections
```css
.page {
  padding: 20px 30px 60px 30px;
  position: relative;
  box-sizing: border-box;
  margin-bottom: 40px;  /* ✅ Visual spacing */
}

.page:last-child {
  margin-bottom: 0;  /* ✅ No trailing space */
}
```

### Table Pagination
```css
thead {
  display: table-header-group;  /* ✅ Repeat on each page */
  break-inside: avoid;
}

tbody tr {
  break-inside: avoid;  /* ✅ Keep rows together */
  page-break-inside: avoid;
}
```

### Section Breaks
```css
.section {
  margin-bottom: 32px;  /* ✅ Visual spacing */
  break-inside: avoid;  /* ✅ Keep together if possible */
}

h2, h3 {
  break-after: avoid;  /* ✅ Don't orphan headings */
  page-break-after: avoid;
}
```

### HTML Structure
```html
<!-- ✅ Clean structure, no inline headers/footers -->
<div class="page">
  <h2>Section Title</h2>
  <p>Content here...</p>
</div>

<div class="page">
  <h2>Next Section</h2>
  <p>More content...</p>
</div>
```

## ❌ DON'T - Problematic Patterns

### Forced Page Breaks
```css
/* ❌ DON'T: Creates blank pages */
.page {
  page-break-after: always;
}

/* ❌ DON'T: Forces breaks everywhere */
.page-break {
  page-break-after: always;
  break-after: page;
}
```

### Hardcoded Page Numbers
```html
<!-- ❌ DON'T: Static page numbers -->
<footer>
  <div>Page 2</div>
</footer>

<!-- ❌ DON'T: Wrong on every page -->
<div>Page 2 of 10</div>
```

### Inline Headers/Footers
```html
<!-- ❌ DON'T: Conflicts with CSS -->
<div class="page">
  <header>
    <div class="generated-by">Generated by VerifyTrade</div>
  </header>
  <p>Content</p>
  <footer>
    <div>Page 2</div>
  </footer>
</div>
```

### Height Constraints
```css
/* ❌ DON'T: Creates empty pages */
.page {
  min-height: 100vh;
  height: 100vh;
}
```

### Inconsistent Margins
```css
/* ❌ DON'T: Different margins per report */
@page {
  margin: 20mm;  /* Some reports */
}

@page {
  margin: 15mm 12mm;  /* Other reports */
}

/* ✅ DO: Use consistent margins */
@page {
  margin: 18mm 14mm 18mm 14mm;  /* All reports */
}
```

## Print CSS Essentials

```css
/* Color preservation */
* {
  print-color-adjust: exact !important;
  -webkit-print-color-adjust: exact !important;
}

/* Hide elements in print */
@media print {
  .no-print {
    display: none !important;
  }
}

/* Keep cards/sections together */
.card, .section-card {
  break-inside: avoid;
  page-break-inside: avoid;
}

/* Prevent orphans/widows */
p, li {
  orphans: 3;
  widows: 3;
}
```

## Gotenberg Page Numbering

Gotenberg automatically numbers pages when:
- ✅ No hardcoded numbers in HTML
- ✅ Footer space reserved via CSS
- ✅ Clean HTML structure

**It injects:** `Page 1 of 5`, `Page 2 of 5`, etc.

## Common Issues & Solutions

| Issue | Cause | Solution |
|-------|-------|----------|
| Blank pages | `page-break-after: always` | Remove forced breaks |
| Wrong page numbers | Hardcoded HTML | Remove static numbers |
| Overlapping text | Inline headers/footers | Remove inline elements |
| Inconsistent margins | Different @page rules | Standardize to 18mm 14mm |
| Table splits badly | No break controls | Add `break-inside: avoid` to rows |

## Testing Checklist

- [ ] No blank pages between sections
- [ ] Page numbers sequential (1, 2, 3...)
- [ ] No "Page 2" on cover
- [ ] Tables break cleanly
- [ ] Headers repeat on new pages
- [ ] No content overlaps headers/footers
- [ ] Consistent margins across reports
