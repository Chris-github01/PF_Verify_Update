# Pricing Basis Dropdown Fix - Root Cause Found!

## The Problem

The "Pricing Basis" dropdown appeared frozen on "Select Pricing Basis..." and wouldn't allow selection of any option. This was **blocking the entire Pre-let Appendix PDF export** because:

1. Users couldn't select a pricing basis
2. Without a valid pricing basis, the appendix couldn't be finalized
3. Without finalization, the PDF couldn't be generated

## Root Cause

**Value Mismatch Bug** (Line 3276 of ContractManager.tsx)

```typescript
// BEFORE (BROKEN):
pricing_basis: existingAppendix?.pricing_basis || 'lump_sum',
```

The default value was `'lump_sum'`, but the dropdown options used different values:
- `fixed_price_lump_sum`
- `fixed_price_lump_sum_quoted_quantities`
- `fixed_price_lump_sum_remeasurable`
- `schedule_of_rates`
- `guaranteed_maximum_price`
- `cost_reimbursable`

Since `'lump_sum'` doesn't match any valid option, the dropdown showed "Select Pricing Basis..." but was internally set to an invalid value. React's controlled component behavior prevented the dropdown from changing.

## The Fix

### Change 1: Initialize with Empty String
```typescript
// AFTER (FIXED):
pricing_basis: existingAppendix?.pricing_basis || '',
```

This ensures the dropdown starts in a truly unselected state, forcing users to explicitly choose an option.

### Change 2: Fix Award Overview Default
```typescript
// BEFORE:
awarded_pricing_basis: 'lump_sum',

// AFTER:
awarded_pricing_basis: 'fixed_price_lump_sum',
```

Changed the default to match the most common valid option value.

## Why This Affected PDF Export

The PDF export flow requires:
```
User selects pricing basis
  ↓
Save appendix with pricing_basis field
  ↓
Finalize appendix (validates all required fields)
  ↓
Generate PDF (includes pricing basis clause)
  ↓
Download PDF
```

With the dropdown broken, users couldn't complete step 1, which blocked the entire workflow.

## Additional Notes

### Database Values
The `pricing_basis` field in the database accepts these enum values:
- `fixed_price_lump_sum`
- `fixed_price_lump_sum_quoted_quantities`
- `fixed_price_lump_sum_remeasurable`
- `schedule_of_rates`
- `guaranteed_maximum_price`
- `cost_reimbursable`

### PDF Integration
Each pricing basis has a corresponding clause that gets injected into the Pre-let Appendix PDF document. These clauses are generated by the `getPricingBasisClause()` function in `contractPrintEngine.ts`.

### Empty String Handling
If `pricing_basis` is empty/null, the PDF generation will work but won't include a pricing basis clause. The dropdown validation should encourage users to select one before finalizing.
