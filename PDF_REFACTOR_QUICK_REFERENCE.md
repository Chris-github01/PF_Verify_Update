# PDF Refactor Quick Reference

## What Changed?

### Visual Changes (User-Facing)
1. **Per-row checkmarks removed** → **Section-level badges added**
2. **Inconsistent headers/footers** → **Unified "Page X of Y" system**
3. **Split table headers** → **Headers repeat on every page**
4. **Orphan pages** → **Smart pagination prevents empty pages**
5. **One-size-fits-all** → **Two themes: Senior (dense) vs Junior (spacious)**

### Code Changes (Developer-Facing)
1. **New files**: `pdfPaginationEngine.ts`, `pdfThemes.ts`, `pdfHeaderFooter.ts`
2. **Modified**: `contractPrintEngine.ts` (CSS + layout only)
3. **Unchanged**: All data logic, queries, calculations

## Using the New System

### Generating PDFs (No API Changes)

```typescript
import { generateContractPDF } from '@/lib/reports/contractPrintEngine';

// Generate Senior Pack (high density, professional)
```

## Troubleshooting

### Issue: Page numbers not showing
**Cause**: JavaScript not executing in PDF renderer
**Fix**: Ensure Gotenberg or print environment supports JS

### Issue: Headers not repeating
**Cause**: CSS `display: table-header-group` not supported
**Fix**: Check browser/renderer version (needs modern CSS)

### Issue: Icons still showing in table rows
**Cause**: Old cached CSS
**Fix**: Hard refresh browser (Ctrl+Shift+R)

### Issue: Wrong theme applied
**Cause**: Pack type mismatch
**Fix**: Verify `packType` parameter: 'site_team' vs 'senior_mgmt'

## Migration Notes

### Before (Old System)
- Single layout for all packs
- Per-row icons in every table
- Inconsistent page numbering
- Tables could split mid-header
- Manual "Generated by" text everywhere

### After (New System)
- Theme-based layouts (2 options)
- Section-level badges only
- Unified "Page X of Y" footer
- Smart pagination prevents splits
- Single source of truth for branding

### Breaking Changes
**NONE** - API unchanged, data unchanged, output structure unchanged

### Deprecations
**NONE** - All existing code continues to work
